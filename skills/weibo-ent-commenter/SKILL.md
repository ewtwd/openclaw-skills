# Weibo Ent Commenter

## Overview

生成一条像老微博用户随手发出来的文娱评论：短、活、情绪到位，不写成客服腔或总结腔。

默认输出 1 条评论，并在以下三种视角里随机选择一种：死忠粉/妈粉、路人/剧粉、乐子人/玩梗。

## Quick Start

- 这是一个**只给 agent 读提示词并现生成评论**的技能
- 风格提示词参考：`references/prompt.md`
- 最终用于微博互动的评论，必须由 agent 读取提示词后结合当前帖子内容即时生成，不要调用本地脚本来产出最终评论

## Rules

执行时遵守这些约束：

1. 把评论控制在微博常见热评语感内，整体保持 10-30 字。
2. 禁止生成书面总结腔，如“综上所述”“总而言之”“作为一名观众”“值得一提的是”。
3. 默认不加句号，不写长段分析，不写像任务说明一样的句子。
4. 允许自然使用语气词、夸张词、emoji、微博文字表情，但不要堆砌。
5. 评论必须贴内容：自拍就夸状态，影视就聊角色/预告/氛围，综艺就聊笑点，吃瓜就偏围观/抖机灵。
6. 如果输入是空、脏、信息极少，仍然给出一条自然兜底评论，但不要暴露“不知道内容”。

## Integration Notes

把这个技能当成微博评论链路的默认文案源：

- agent 在处理“链接 评论 / 链接 转发与评论 / 点赞后评论”这类任务时，应先读取本技能与 `references/prompt.md`，再基于当前帖子内容现生成 1 条评论
- 然后把生成结果通过 `social-publisher/scripts/interact-weibo.ps1 -Text ...` 传给执行脚本
- 不要依赖执行脚本内部模板来生成评论
- 不要调用 `scripts/run-weibo-ent-commenter.mjs` / `.ps1` 或 `weibo-ent-commenter.mjs` 来产出最终评论；这些脚本至多用于本地测试，不用于正式执行链路
